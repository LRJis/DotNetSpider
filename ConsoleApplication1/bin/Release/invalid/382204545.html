关系运算符： <、 >、 <=和 >=

  • 项目
  • 07/19/2022

本文内容

语法

expression < expression
expression > expression
expression <= expression
expression >= expression

备注

二进制关系运算符确定下列关系：

  • 小于号 (<)

  • 大于 (>)

  • 小于或等于 (<=)

  • 大于或等于 (>=)

关系运算符具有从左到右的关联性。 关系运算符的两个操作数必须是算术或指针类型。 它们会生成类型的 bool值。 如果表达式中的关系为 false，则返回 false 的值 (0) ;否则，返回的值 (true 1) 。

示例

// expre_Relational_Operators.cpp
// compile with: /EHsc
#include <iostream>

using namespace std;

int main() {
   cout  << "The true expression 3 > 2 yields: "
         << (3 > 2) << endl
         << "The false expression 20 < 10 yields: "
         << (20 < 10) << endl;
}

前面的示例中的表达式必须括在括号中，因为流插入运算符 () << 的优先级高于关系运算符。 因此，未括在括号中的第一个表达式的计算结果将为：

(cout << "The true expression 3 > 2 yields: " << 3) < (2 << "\n");

标准转换中介绍的常用算术转换应用于算术类型的操作数。

比较指针

在比较两个指向同一类型的对象的指针时，结果由程序的地址空间中所指向的对象的位置决定。 指针也可以与计算结果为 0 或类型的 void *指针的常量表达式进行比较。 如果针对类型的 void *指针进行了指针比较，则另一个指针将隐式转换为类型 void *。 然后进行比较。

不能比较两个类型不同的指针，除非：

  • 一个类型是派生自另一个类型的类类型。

  • 将至少一个指针显式转换为类型 void * (强制转换) 。 (将另一个指针隐式转换为转换类型 void * 。)

两个指向同一对象的相同类型的指针一定是相等的。 如果比较两个指向对象的非静态成员的指针，则以下规则将适用：

  • 如果类类型不是， union并且两个成员不是 由访问说明符（例如 public） protected分隔的，或者 private，最后声明的成员的指针将比较大于前面声明的成员的指针。

  • 如果两个成员由 访问说明符分隔，则结果未定义。

  • 如果类类型是一个 union，则指向该比较中不同数据成员的 union 指针相等。

如果两个指针指向同一数组的元素或指向超出数组末尾 1 的元素，则指向带较高下标的对象的指针会更高。 仅当指针引用同一数组中的对象或超出数组末尾 1 的位置时，才能保证指针比较有效。
